100% Local & Private
Raspador runs entirely inside your browser. No data ever leaves your device. No servers. No accounts. No telemetry.
What Stays on Your Device
Extracted data
All rows collected from pages are held in browser memory only. Nothing is written to any external database or service.
Saved configs
Site configurations (selectors, XPath columns, filters) are stored in Chrome's local storage on your machine — the same place bookmarks live. They never sync to Google unless you have Chrome Sync turned on for extensions.
CSV exports
Files are written directly to your Downloads folder via Chrome's built-in download API. No third party touches them.
Network requests
Raspador makes zero network requests of its own. The only traffic generated is your browser's normal page navigation during crawling — identical to clicking Next yourself.
Technology & Components
Chrome Extensions APIManifest V3
Google Chrome — open specification. The official API for browser extensions. Governs permissions, messaging, downloads, and storage.
Content ScriptsVanilla JS
Written entirely from scratch for Raspador. Injected into pages only when the extension is active. Reads the DOM — never modifies page content or sends data out.
Popup UIHTML / CSS / JS
Hand-written. No UI framework, no npm packages, no external dependencies. The entire interface is the three files bundled in this extension.
XPath Evaluationdocument.evaluate()
Native browser API (W3C standard). Built into every browser since IE 5. No library required — zero external code.
Regex FilteringJS RegExp
Standard JavaScript regular expressions, built into the V8 engine that powers Chrome. No external library.
CSV ExportBlob + chrome.downloads
File is constructed in memory as a UTF-8 string with BOM (for Excel compatibility), then handed to Chrome's native download manager. No server upload, no cloud storage.
Config Storagechrome.storage.local
Chrome's sandboxed local key-value store. Data lives on your machine at the OS profile level. Only this extension can read or write it.
IconsPNG — generated locally
Generated with Python + Pillow (open-source imaging library). No icon fonts, no CDN, no external image service.
Permissions Explained
activeTab
Lets Raspador read the DOM of the tab you are currently viewing — only when you click the extension icon. It cannot access background tabs.
scripting
Required to inject the content script that detects and extracts data. The script is read-only with respect to page content.
downloads
Allows saving CSV files to your Downloads folder without showing a browser "Save As" dialog every time.
storage
Used exclusively to save and load your site configurations locally. No cloud sync unless you have enabled it in Chrome settings.
<all_urls>
The content script needs permission to run on any site you navigate to. Without this it could only work on a pre-approved list of domains. Raspador uses this solely to read page structure — it does not transmit any content from any URL.
